# VALLEY — 二十章云海之旅

一个用 Three.js 制作的《纪念碑谷》风格网页游戏:云海中的三座高塔、
等距视角、柔和渐变配色,以及贯穿全程的核心机制——**旋转空间,改变连通**。

当前版本已经从单章原型扩展为 **20 章连续进度**:每章有独立名称、目标提示、
起点、终点、机关初始朝向和镜头落点;通关后进入下一章,本地保存当前进度。

## 在线试玩

https://chocolatemale.github.io/valley-silent-spire/

## 运行

需要本地静态服务器(ES Module + CDN 依赖):

```bash
cd "Monument Valley"
python3 -m http.server 8642
# 打开 http://localhost:8642
```

依赖 jsdelivr CDN 上的 three@0.180.0,首次加载需联网。

## 操作

- **点击地面**:角色寻路走过去(不可达的位置会显示灰色标记)。
- **点击转盘**:对应颜色的机关旋转 90°(青色转盘 ↔ 长桥,玫瑰转盘 ↔ 悬空楼梯)。
- **光印**:中后期章节需要先收集漂浮光印,金色目标才会打开。
- **金色光环**:当前章节的目标点,踩上去即可进入下一章。
- **右上角星路**:显示 20 章解锁进度,已点亮章节可随时回放。
- 站在旋转件上时转动转盘,**人会随空间一起转过去**。

## 章节进度

20 章围绕同一座云海塔群逐步展开:

1. 前 6 章:学习行走、低桥、主塔楼梯和西露台。
2. 第 7-10 章:围绕青色长桥和蓝绿传送门做空间骑乘。
3. 第 11-17 章:进入高空回廊、玫瑰悬空楼梯和峰顶神龛。
4. 第 18-20 章:把前面的两个机关、传送门和最终金门串成更长路线。

## 实现要点

- `index.html` + `main.js`,无构建步骤。
- 20 章数据位于 `main.js` 的 `CHAPTERS` 数组,包含章节名、起点、终点、
  两个旋转机关初始角度、镜头和目标提示。
- 章节光印位于 `CHAPTER_SIGILS`,共 30 个可收集目标,用于拉长路线和制造回看。
- 行走图:每块地砖/楼梯有"接口点"(port),两节点接口点重合即连通;
  旋转件节点的世界坐标与楼梯朝向随旋转重算,**旋转 = 改变图的边**,BFS 寻路。
- 传送门:成对的门节点,走入一扇即淡出、从另一扇淡入走出(双向)。
- 进度:通关后进入下一章,`localStorage` 保存当前章节与最高解锁进度。
- 视觉:顶点色烘焙(逐面明暗 + 高度渐变)+ Lambert + 长投影 +
  GTAO + Bloom + 调色 Pass(饱和度/暗部偏紫/暗角/胶片颗粒)。
- 音效:WebAudio 即时合成的五声音阶提示音(转盘/落定/传送/通关)。
- 调试接口:控制台 `game.walkTo(id)` / `game.tapDial(0|1)` / `game.skipIntro()`
  / `game.step(dt)`(确定性单步,供自动化测试)。
